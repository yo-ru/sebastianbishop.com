gulag-web

gulag-web, now guweb, is the front-facing appearance of the osu! server procol, gulag! Using native async/await syntax written on top of Quart and cmyui's multipurpose library, guweb achieves flexability, cleanliness, and efficency not seen in other frontend implementations - all while maintaining the simplicity of Python.

gulag

gulag is cmyui's implementation of an osu! server's backend; it handles connections from the osu! client, and has a developer REST API for programmatic interaction.

It's asynchronous design allows it to very efficently manage the IO overhead of an osu! private server (many external requests to the osu!api, a beatmap mirror, database, etc.), and it implements much more effective caching than any other competitive implementation. It's written in modern, high-level Python from the transport (TCP/IP) socket layer directly using cmyui's lightweight web framework.

Cmyui aims to make this project the ideal choice for running osu! private servers, in terms of it's featureset, efficency, safety, and development ease and simplicity. When gulag becomes production-ready, it will be used on Akatsuki which is currently the most active private server; it should be the ideal test.

Switchless

Switchless is a modern osu! private server switcher using the newly implemented "-devserver" startup variable. A osu! server switcher allows the end user to swap back and forth between multiple osu! private servers. Each osu! private server has different credentials and userbases.

Originally osu! private server owners would have to forge SSL certificates that the end user was required to install on their system. This created many security issues if the server owner had any malicious intent, not to mention the legal gray area all of these server owners were jumping into by forging these certificates. Now the end user just has to start osu! using a shortcut with the preappended variable following the servers domain name. An example would most likely look like "path/to/osu!.exe" -devserver example.com. This variable is then used by the client to change all the endpoint URLs used by the client allowing the private server owner to no longer require these illicit SSL certificates.

While Switchless is very barebones. It can run on any Windows System without requiring any libraries to be installed as it only relies on classes and functions within the Windows System itself.

Sekai

Sekai is my personal Discord bot written in modern Python! Using the latest feature set of Discord's API, Sekai is capable of using slash commands. This allows for a more interpersonal experience with the end user via autocompletion of commands as well as descriptions being provided for each command and it's parameters. Like many of my other projects, Sekai is written on top of cmyui's multipurpose library. This allows for the codebase to be efficent as well as developer friendly.

Sekai is still pretty barebones and is lacking functionality. As Sekai grows I will be sure to update it's command set here.

/play <URL/Search> - Plays a requested video/song, using YouTube DL, in the currently connected voice channel.
/stop - Stops the currently playing video/song in the currently connected voice channel.
/lookup <profile> <mode> - Look up user statistics for a specified osu! profile.
/ping - Get an accurate representation of latency between Sekai and Discord.
/uptime - Get Sekai's current total operation time.
/version - Get Sekai's current development version.
/shutdown - Shutdown Sekai. (Requires ownership.)
